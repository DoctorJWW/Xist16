\doxysubsection{Detailed Description}
Graphics functionality. 

Contains macros, structs, extern variables, and function signatures for Xist graphics functionality generally and sprites in particular.

\doxysubsubsection{\texorpdfstring{XIST\_MAX\_SPRITES}{XIST\_MAX\_SPRITES}}
{\footnotesize\ttfamily \#define XIST\+\_\+\+MAX\+\_\+\+SPRITES~64}



The maximum number of sprites supported. 

Defines the maximum number of sprites your application will support and controls the size of {\ttfamily xist\+\_\+sprite\+\_\+bank}. The maximum possible value for this macro is 128 because that is the maximum number of sprites supported by the Commander X16. You should not set this value higher than necessary, because the higher you set this value, the more memory will be consumed, and processing more sprites will entail a greater processor strain. \Hypertarget{xist__gfx_8h_a126288d7b73bbe5db24ba168b830dbd7}\label{xist__gfx_8h_a126288d7b73bbe5db24ba168b830dbd7}

\doxysubsubsection{\texorpdfstring{XIST\_SPRITE\_FINISHED}{XIST\_SPRITE\_FINISHED}}
{\footnotesize\ttfamily \#define XIST\+\_\+\+SPRITE\+\_\+\+FINISHED~0b00000100}



Bit in the sprite bitmask that determines if it the animation is finished playing. 

This value is only relevant to sprites that do not repeat, and will be set to 1 once the non-\/repeating sprite is finished animating.

Readies a sprite which has been loaded into high-\/\+RAM for display and animation. 

Prepares a sprite whose graphical data has already been loaded into high-\/\+RAM for display and animation. It sets up all the necessary parameters for the sprite using the given index into {\ttfamily xist\+\_\+sprite\+\_\+bank} (which is also the same as the sprite\textquotesingle{}s index in the CX16\textquotesingle{}s sprite metadata). Can also be directed to copy the sprite\textquotesingle{}s graphical data from high-\/\+RAM into VRAM.


\begin{DoxyParams}{Parameters}
{\em metadata\+\_\+index} & The row in your graphics metadata file that contains the information for this sprite. \\
\hline
{\em copy\+\_\+to\+\_\+vram} & Set to TRUE if you want to copy the image data from high-\/\+RAM into VRAM. The use case for setting this to FALSE is if you have many sprites that use the same image data, in which case you could set this to TRUE when preparing the first sprite and FALSE for all subsequent calls to save a lot of unnecessary copying. \\
\hline
{\em vram\+\_\+address} & If loading to VRAM, supply the VRAM address to load into with this parameter. Ignored if to\+\_\+ vram is set to FALSE. \\

\hline
